Attack Scenario

An attacker could exploit this vulnerability to:

  • Bypass authentication mechanisms

  • Modify or delete data

Evidence

HTTP Response:

HTTP/1.1 302

Location: https://github.com/juice-shop/juice-shop
Server: Heroku

Found. Redirecting to https://github.com/juice-shop/juice-shop

Impact

Successful exploitation could lead to:

  • Unauthorized database access
  • Sensitive data exposure
  • Data modification or deletion
  • System compromise

Recommendation

Validate redirect URLs against a whitelist of allowed domains.

// Safe Redirect:
$allowed_domains = ['example.com', 'sub.example.com'];
$redirect_url = $_GET['url'];
$parsed = parse_url($redirect_url);
if (in_array($parsed['host'], $allowed_domains)) {
    header("Location: " . $redirect_url);
} else {
    header("Location: /");
}
  • Avoid using user input directly in redirect headers
  • Use relative paths when possible
